SpringCloud

单体应用

  1. 什么是单体应用

    所有的功能都在一个包里面

  2. 单体应用的缺点：

    • 单节点问题
    • 项目会越来越大，编译，维护麻烦，责任不分离
    • 性能问题，局部拓展不方便
    • 重启，全部都要重启
    • 技术选型单一
    • 数据库选型单一

微服务架构

  1. 什么是微服务

    就是把一个项目拆分成多个小的服务(应用)，每个服务都有自己独立的tomcat，有自己独立的进程，有自己独立的数据库，一个服务只关注一个功能，服务与服务之间使用网络协议进行通信

  2. 微服务的特点：

    • 微服务架构由多个小的服务注册

    • 每个服务都有自己的进程

SpringCloud简介

  1. 什么是springcloud

    Spring Cloud是一个含概多个子项目的开发工具集,集合了众多的开源框架,他利用了Spring Boot开发的便利性实现了很多功能,如服务注册,服务注册发现,负载均衡等.Spring Cloud在整合过程中主要是针对Netflix(耐非)开源组件的封装.

  2. 常见的组件

    • 服务发现——Netflix Eureka
    • 客服端负载均衡——Netflix Ribbon
    • 断路器——Netflix Hystrix
    • 服务网关——Netflix Zuul
    • 分布式配置——Spring Cloud Config

  3. springcloud和dubbo的区别

    dubbo只解决网络通信问题

    springcloud是微服务框架解决一系列问题，基于springboot，使用起来简单

ElasticSearch

1. ElasticSearch的基本概念

ElasticSearch是一个基于Lucene的搜索服务器。它提供了一个分布式多用户能力的全文搜索引擎，基于RESTful web接口。Elasticsearch是用Java语言开发的，并作为Apache许可条款下的开放源码发布，是一种流行的企业级搜索引擎。ElasticSearch用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。官方客户端在Java、.NET（C#）、PHP、Python、Apache Groovy、Ruby和许多其他语言中都是可用的。

2. ES的特点

  • Es 全文检索框架 使用比lucene更简单
  • ES支持集群 支持分布式
  • 支持json操作
  • 一般大型全文检索都用ES

  • 通过Restfull风格操作ES

    3. Es和Lucene的区别

    Lucene：

  • 使用起来很复杂

  • Lucene不支持集群
  • 扩展麻烦
  • 必须使用Java

Nosql

1. Nosql的概念

NoSql指非关系型数据库，性能高，解决高并发问题

2. Nosql的分类

  • key-value键值对数据库
  • 列式存储数据库
  • 文档型结构数据库
  • 图形结构数据库

    Redis介绍

    1. Redis的概念

    Redis是属于Key-Value键值对数据库，是Nosql的一种

    2. Redis的特点

  • 数据存储到内存
  • 支持数据持久
  • 支持Redis集群
  • 支持事务
  • 支持订阅发布

    3. Redis的使用场景

  • 中央缓存
  • 计数器
  • 实时防攻防系统
  • 排行榜
  • 有效时间
  • 去重
  • 队列
  • 发布订阅

    Redis的安装

    Redis的常用命令

    1. String的基本操作

    设置值：set key value
    取值：get key
    同时设置一个或多个 key-value 对：mset
    返回所有(一个或多个)给定 key 的值：mget
    将 key 中储存的数字值增1(key不存在,则初始化为0,再加1)：incr key
    将 key 中储存的数字值减1(key不存在,则初始化为0,再减1)：decr key
    自增多少：incrBy key num
    自减多少：decrBy key num

    2. Key的命令

    获取所有key列表：key *
    删除key：del key
    设置key的过期时间：expire key xx
    查看key的过期时间：ttl key
    选择库 默认16个数据库：select 0-15
    清空整个redis服务器数据，所有的数据库全部清空：flushall
    清除当前库，redis中默认有16个数据库，名称分别为0,1,2.。。15：flushdb
