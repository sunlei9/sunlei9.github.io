SpringCloud

单体应用

  1. 什么是单体应用

    所有的功能都在一个包里面

  2. 单体应用的缺点：

    • 单节点问题
    • 项目会越来越大，编译，维护麻烦，责任不分离
    • 性能问题，局部拓展不方便
    • 重启，全部都要重启
    • 技术选型单一
    • 数据库选型单一

微服务架构

  1. 什么是微服务

    就是把一个项目拆分成多个小的服务(应用)，每个服务都有自己独立的tomcat，有自己独立的进程，有自己独立的数据库，一个服务只关注一个功能，服务与服务之间使用网络协议进行通信

  2. 微服务的特点：

    • 微服务架构由多个小的服务注册

    • 每个服务都有自己的进程

SpringCloud简介

  1. 什么是springcloud

    Spring Cloud是一个含概多个子项目的开发工具集,集合了众多的开源框架,他利用了Spring Boot开发的便利性实现了很多功能,如服务注册,服务注册发现,负载均衡等.Spring Cloud在整合过程中主要是针对Netflix(耐非)开源组件的封装.

  2. 常见的组件

    • 服务发现——Netflix Eureka
    • 客服端负载均衡——Netflix Ribbon
    • 断路器——Netflix Hystrix
    • 服务网关——Netflix Zuul
    • 分布式配置——Spring Cloud Config

  3. springcloud和dubbo的区别

    dubbo只解决网络通信问题

    springcloud是微服务框架解决一系列问题，基于springboot，使用起来简单

ElasticSearch

1. ElasticSearch的基本概念

ElasticSearch是一个基于Lucene的搜索服务器。它提供了一个分布式多用户能力的全文搜索引擎，基于RESTful web接口。Elasticsearch是用Java语言开发的，并作为Apache许可条款下的开放源码发布，是一种流行的企业级搜索引擎。ElasticSearch用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。官方客户端在Java、.NET（C#）、PHP、Python、Apache Groovy、Ruby和许多其他语言中都是可用的。

2. ES的特点

  • Es 全文检索框架 使用比lucene更简单
  • ES支持集群 支持分布式
  • 支持json操作
  • 一般大型全文检索都用ES

  • 通过Restfull风格操作ES

    3. Es和Lucene的区别

    Lucene：

  • 使用起来很复杂

  • Lucene不支持集群
  • 扩展麻烦
  • 必须使用Java

    1. String的基本操作

    设置值：set key value

    返回所有(一个或多个)给定 key 的值：mget
    将 key 中储存的数字值增1(key不存在,则初始化为0,再加1)：incr key

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

面试题

  1. forword和redirect的区别以及各自的应用场景
    forword请求转发，redirect重定向
    区别：
    1. forward是一个请求的延续，可以共享request的数据，redirect开启一个新的请求，不可以共享request
2. forward地址栏不发生变化，redirect地址栏发生变化
3. forward是发送一次请求，redirect是发送两次请求
4. forward不能跨域访问，redirect能跨域访问
5. forward请求能访问WEB-inf下main的资源
    应用场景
    forward一般用于用户登录的时候，根据角色转发到相应的模块
    redirect一般用于用户注销登录时返回主页面和跳转到其他的网站等
  2. 为什么要进行事务管理,Spring是如何进行事务管理支持的
